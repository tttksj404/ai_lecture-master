# Day 8: AI 데이터 생성 — 초간단 요약

처음 배우는 사람 기준으로, 필요한 내용만 쉽게 정리했습니다.

---

## 1) 이 수업에서 하는 일 (한 줄 요약)

**LLM으로 새 데이터를 만들고, 품질을 검사해서, 좋은 데이터만 골라 쓰는 방법을 배웁니다.**

---

## 2) 꼭 알아야 할 단어 5개

- **데이터 증강**: 기존 데이터를 변형해서 양을 늘리는 것  
- **합성 데이터**: 실제 수집 없이 AI가 새로 만든 데이터  
- **Zero-shot**: 예시 없이 지시만 주는 방식  
- **Few-shot**: 예시를 몇 개 보여주고 생성시키는 방식  
- **LLM as Judge**: LLM이 생성 결과 품질을 점수로 평가하는 방식  

---

## 3) 실습 흐름 (쉽게)

1. **API 연결**  
   - `.env`에 키를 넣고 LLM 호출 준비

2. **프롬프팅 비교**  
   - Zero-shot / Few-shot / CoT 결과 차이 확인

3. **구조화된 데이터 생성**  
   - JSON 형식으로 정리된 출력 만들기

4. **품질 평가**  
   - LLM as Judge로 점수 매기고 필터링

---

## 4) 데이터 증강 vs 합성 데이터

- **증강**: 원래 데이터를 바꿔서 다양하게 만듦
- **합성**: 원래 데이터 없이 새로 만들어 냄

쉽게 말해:
- 증강 = \"원본 변형\"
- 합성 = \"새로 제작\"

---

## 5) 왜 LLM as Judge가 필요할까?

- 생성 데이터는 품질이 들쭉날쭉함
- 자동 점수화로 빠르게 나쁜 샘플 제거 가능
- 좋은 데이터만 남겨서 학습 품질 향상

---

## 6) 실수하기 쉬운 포인트

- API 키를 코드에 직접 쓰지 말고 `.env` 사용
- 평가는 `temperature=0`으로 일관성 확보
- 같은 모델로 생성+평가하면 편향 가능성 있음

---
