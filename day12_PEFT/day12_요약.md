# Day 12: 효율적인 Fine-tuning (PEFT) — 초간단 요약

처음 배우는 사람 기준으로, 핵심만 쉽게 정리했습니다.

---

## 1) 이 수업에서 하는 일 (한 줄 요약)

**큰 모델을 통째로 다시 학습하지 않고, 필요한 부분만 똑똑하게 튜닝하는 방법(PEFT)을 배웁니다.**

---

## 2) 꼭 알아야 할 단어 5개

- **Fine-tuning**: 이미 학습된 모델을 내 목적에 맞게 추가 학습하는 것
- **PEFT**: 적은 파라미터만 학습해서 비용을 줄이는 튜닝 방식
- **LoRA**: 저차원 행렬을 추가해 적은 비용으로 성능을 올리는 방법
- **QLoRA**: 모델을 4-bit로 줄인 뒤 LoRA를 적용하는 방식
- **양자화(Quantization)**: 숫자 표현 비트를 줄여 메모리를 아끼는 기술

---

## 3) 왜 PEFT가 필요할까?

- Full Fine-tuning은 모델 전체를 업데이트해서 GPU 메모리를 많이 사용함
- 실제 현업에서는 시간/비용/장비 제약이 큼
- PEFT는 "필요한 부분만" 학습해서 속도와 비용을 절약함
- 작은 자원으로도 맞춤형 모델을 만들 수 있음

쉽게 말해:
- Full FT = 집 전체를 다시 공사
- PEFT = 필요한 방만 리모델링

---

## 4) 실습 흐름 (쉽게)

1. **모델 준비**
   - Unsloth/HuggingFace로 기본 모델과 토크나이저를 불러옴

2. **메모리 비교**
   - 32bit, 16bit, 8bit, 4bit에서 얼마나 메모리를 쓰는지 확인

3. **LoRA 설정**
   - rank, alpha, dropout 같은 핵심 값을 정함

4. **데이터 전처리**
   - Chat Template로 입력 형식을 맞추고 label masking 적용

5. **학습/추론 확인**
   - 파인튜닝 후 답변 품질과 자원 사용량을 함께 점검

---

## 5) 핵심 개념 비교

- **Full Fine-tuning**
  - 장점: 모델 전체를 직접 최적화
  - 단점: 메모리/시간 비용이 매우 큼

- **LoRA**
  - 장점: 학습 파라미터 수가 적어 효율적
  - 단점: 설정값에 따라 성능 변동이 큼

- **QLoRA**
  - 장점: 메모리 절감 효과가 커서 제한된 GPU에 유리
  - 단점: 양자화로 인해 세밀한 품질 튜닝이 필요할 수 있음

---

## 6) 실수하기 쉬운 포인트

- LoRA를 적용할 target module을 잘못 지정함
- 학습률을 너무 크게 잡아 출력이 불안정해짐
- Chat Template 형식이 모델과 맞지 않아 성능이 떨어짐
- Label masking을 안 해서 질문까지 그대로 외우는 문제가 생김
- 성능만 보고 메모리/속도 지표를 같이 보지 않음
